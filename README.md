## 🎮 游戏简介

《杀戮尖塔·简易版》是南京大学《程序设计基础》课程的大作业成果。  
游戏采用 **C11 标准** 编写，基于自建的 `sts_io` 控制台 I/O 库，实现了 **卡牌抽取、能量管理、敌人 AI、状态效果（Buff/Debuff）、多阶段战斗** 等核心机制。  
你可以通过 **主菜单** 进入战斗，利用 **打击、防御、放血、暴走** 等卡牌与敌人周旋，体验策略与运气并存的肉鸽乐趣。

---

## 🧩 核心功能

- ✅ **主菜单与回合循环** — 清晰的标题界面、主菜单和战斗循环，支持反复游玩。
- ✅ **HUD 战斗界面** — 实时显示玩家与敌人的生命、格挡、能量，以及敌人意图。
- ✅ **卡牌系统** — 基于 `struct Card` 的卡牌表示，支持 **打击、防御、放血、暴走、鲁莽、中毒、传染** 共 7 种卡牌。
- ✅ **动态牌堆** — 抽牌堆、手牌、弃牌堆的完整流转；Fisher-Yates 洗牌算法保证随机性。
- ✅ **能量与费用** — 每回合 3 点能量，卡牌费用不足时无法打出。
- ✅ **敌人多阶段 AI** — 一阶段固定循环（加甲→攻击→复合），半血后进入二阶段，行为模式切换。
- ✅ **状态效果（扩展）** — 实现了 **易伤（受伤+50%）、虚弱（造成伤害-25%）、中毒（每回合扣血）**，且有对应的施加卡牌。
- ✅ **自动化测试** — 使用 `assert` 编写了覆盖核心逻辑的测试集，支持条件编译切换固定/随机种子。

## 🧱 技术亮点

| 技术点 | 说明 |
|--------|------|
| **C 语言面向对象思想** | 通过 `struct` 封装玩家、敌人、卡牌数据，函数指针数组实现卡牌效果分发 |
| **Fisher-Yates 洗牌** | 实现真随机牌堆，保证每次游戏抽牌顺序不同 |
| **静态局部变量** | 封装随机数生成器 `init_rand()`，确保种子只初始化一次 |
| **状态效果系统** | 易伤、虚弱、中毒三种 Buff/Debuff，伤害计算采用修正流水线 `calc_damage()` |
| **多阶段敌人** | 敌人状态机管理一阶段、过渡、二阶段，意图实时更新 |
| **自动化测试** | 数十条 `assert` 覆盖牌堆操作、战斗逻辑、状态效果，支持固定种子重现 |
| **跨平台兼容** | 纯 C11 标准代码，支持 macOS / Linux / Windows 编译运行 |

## 🚀 快速开始

### 编译运行

本项目使用 **CMake** 构建，推荐使用 CLion 打开。  
最低要求 C11 标准，需要链接 `sts_io` 库（已内嵌在项目中）。

```bash
# 克隆仓库
git clone https://github.com/fangxingzhu/NJU26_spring_Fundamentals_of_Programming_Design_Major_Assignment.git
cd "my project"

# 使用 CMake 构建
mkdir build && cd build
cmake .. -DCMAKE_C_STANDARD=11
make
./sts_io   # 运行游戏
```

或在终端直接编译（确保已包含 `sts_io.c` 和 `sts_io.h`）：

```bash
gcc -std=c11 main.c sts_io.c -o game
./game
```

### 测试模式

如需运行自动化测试，在 `main.c` 首行取消注释：
```c
#define TEST_MODE
```
重新编译运行即可。测试通过则程序无额外输出，否则会断言失败并终止。

## 🃏 卡牌图鉴

| 卡牌名 | 费用 | 效果 |
|--------|------|------|
| 打击 | 1 | 造成 6 点伤害 |
| 防御 | 1 | 获得 5 点格挡 |
| 放血 | 0 | 对自己造成 3 点伤害，回复 1 点能量 |
| 暴走 | 1 | 造成 8 点伤害，每次打出后该牌伤害永久 +5 |
| 鲁莽 | 1 | 造成 12 点伤害，自身获得 **虚弱** |
| 中毒 | 1 | 敌人获得 **5 层中毒**，自身获得 **易伤** |
| 传染 | 0 | 将自身的易伤/虚弱 **复制** 给敌人 |

## 👾 敌人行为

- **一阶段**（HP > 20）：循环执行 “加 6 格挡 → 8 伤害 → 5 伤害 + 10 格挡”
- **二阶段**（HP ≤ 20）：进入时立即造成 **16 点伤害**，之后交替 “加 10 格挡” 与 “12 伤害”

敌人意图会在 HUD 中提前显示，让玩家决策更有依据。
